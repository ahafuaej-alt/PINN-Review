After you submit

A visible five-step publication trail

Every stage appears in the GitHub request, so you can see whether the data were only drafted, submitted, validated, approved, or published.

  1. 1. Confirm on GitHubThe green button opens a prefilled request. Select Create new issue; only then is the information sent.
  2. 2. Automatic validationGitHub checks the package and comments either “Validation passed” or the exact field that must be corrected.
  3. 3. ApproveAfter reviewing the evidence and preview, a repository maintainer comments /apply-dataset-update.
  4. 4. RebuildThe master record, MDPI citation, References, PINN Realm, year totals, collaboration data, exports, version, and audit trail are regenerated together.
  5. 5. Verify publicationThe request receives the new dataset version and closes after the update is committed. GitHub Pages then deploys the synchronized Atlas.
